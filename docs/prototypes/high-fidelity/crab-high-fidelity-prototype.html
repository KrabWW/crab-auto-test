Project Governance Settings

Settings

低频治理入口。顶层只保留 General、Members、Model Providers、Credentials、Audit、Danger Zone；环境变量和使用范围作为对应治理页的子区块。

General

项目身份、默认环境、发布门禁策略。不要把所有治理能力平铺到一个一级页面。

MCP permissions summary

详细审批和 allowlist 在 MCP Governance；Settings 只展示项目级风险摘要。

Knowledge usage summary

Skill enablement summary

Danger Zone

危险操作隔离展示，每个动作都要说明后果、要求角色、确认短语、影响对象和审计结果。

Reset evidence for project

会清空 Evidence Matrix 和报告结论，不删除需求与用例。需要 owner 输入 RESET EVIDENCE。

Revoke all credentials

会停止模型、MCP、API 自动化的外部访问。需要 owner 输入 REVOKE CREDENTIALS。

Delete project

不可恢复。需要导出审计记录后执行。
